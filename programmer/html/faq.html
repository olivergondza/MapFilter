MapFilter

What is MapFilter Pattern?

Pattern is such class that implements MapFilter_Pattern_Interface (and optionally same other of MapFilter_Pattern interfaces). All filtering is going to be performed against this Pattern.

What is MapFilter Query?

Query is a user Key-Value storage intended to filter.

What is MapFilter Result?

Result is a complex outcome of the filtering procedure provided by MapFilter::fetchResult(). This result is provider of the same set of interfaces as given filtering Pattern.

TreePattern

TreePattern is default MapFilter_Pattern_Interface implementation.

Nodes

What is the All node for?

This nodes get satisfied when node specific rules are met. The All node get satisfied in case all it’s followers are satisfied.

What is the One node for?

This nodes get satisfied when node specific rules are met. Only one follower of the One node can be satisfied for the One node to get satisfied.

What is the Opt node for?

This nodes get satisfied when node specific rules are met. The Opt node is satisfied regardless of it’s followers.

What is the Some node for?

This nodes get satisfied when node specific rules are met. The Some node needs at least one satisfied followers to be satisfied itself.

Leaves

What is the Attr leaf?

The Attr leaf is the only leaf of TreePattern.

What is the KeyAttr leaf?

It’s the only TreePattern node that makes decisions considering attribute value. Must have exactly one satisfied follower to be satisfied itself. All it’s followers has to have forValue attribute.

Attributes

What is the forValue attribute?

Attribute forValue is allowed for all TreePattern node elements that are followers of KeyAttr node. Contains regular expression to match by KeyAttr value.

What is the assert attribute?

The assert attribute enables to set assertions in case that attribute or group of attributes fails in satisfaction process. This attribute is allowed for all pattern elements. All sat assertions can be had by getAsserts() from parsed filter.

What is the flag attribute?

The flag is a universal pattern element attribute to provide some additional information from parsing process. When pattern node got satisfied it’s flag will be sat. This flags are accessible by getFlags().

What is the valuePattern attribute?

The valuePattern in an optional attribute of Attr leaf. Defines REGEXP to match for future attribute value. Attr leaf with this pattern can be satisfied only if both attribute is present and it’s value matches pattern.

What is the default attribute?

The default attribute is an optional Attr node attribute. If there is an unsatisfied Attr node with defined default attribute, default attribute will be used and node will be considered satisfied.
