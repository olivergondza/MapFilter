TreePattern Syntax

The pattern is represented by XML file or string with following sintax.

All allowed elements and its attributes are declared in this document.

There are some groups declared to facilitate further declarations.

  • ANY = Any "Tree element".
  • NONE = No element an all.
  • ? = Element is optional.
  • + = One or more elements can be present.
  • * = Any number of elements.

i.e.:

  • ANY* = Any number of any elements.
  • name? = There is an optional attribute called "name".
  • pattern+ = One or more pattern elements.
Note:
These groups are not used besides this document.

Special elements

Patterns element

tag:
<patterns>
attributes:
NONE
children:
pattern+

The <patterns> tag is used as the root element in case there are more patterns in one file or string. Thus, if there is just one <pattern> element this element becomes optional.

<patterns>
  <pattern>
    <attr>MyAttr</attr>
  </pattern>
</patterns>

is the same as 

<pattern>
  <attr>MyAttr</attr>
</pattern>

Pattern element

tag:
<pattern>
attributes:
name?
children:
ANY

The <pattern> tag is used to aggregate the pattern tree itself. In case there is just one pattern in the file this tag become optional.

<pattern>
  <attr>MyAttr</attr>
</pattern>

is the same as 

<attr>MyAttr</attr>

There is an optional attribute called "name" with default value "main". This name is used by attachPattern attribute.

<pattern name="main">
  <attr>MyAttr</attr>
</pattern>

is the same as 

<pattern>
  <attr>MyAttr</attr>
</pattern>

In case there are multiple pattern elements in the file they must have different names assigned.

Tree elements

Elements without children

Pattern specification containts elements that has no children but can have text content.

Attr element

tag:
<attr>
attributes:
forValue?, attachPattern?, assert?, flag?, attr?, valuePattern?, default?, iterator?
children:
NONE

The value of the optional attribute attr can be assigned as the tag content as well.

<attr>MyAttr</attr>

is the same as 

<attr attr="MyAttr" />

Tag content overwrites attribute value.

<attr attr="MyAttr">YourAttr</attr>

is the same as 

<attr attr="YourAttr" />

or 

<attr>YourAttr</attr>

Allowed values of an iterator attribute for <attr> element are "yes", "no" or "auto".

Elements with children

All element

tag:
<all>
attributes:
forValue?, attachPattern?, assert?, flag?
children:
ANY*

One element

tag:
<one>
attributes:
forValue?, attachPattern?, assert?, flag?
children:
ANY*

Opt element

tag:
<opt>
attributes:
forValue?, attachPattern?, assert?, flag?
children:
ANY*

Some element

tag:
<some>
attributes:
forValue?, attachPattern?, assert?, flag?
children:
ANY*

KeyAttr element

tag:
<key_attr>
attributes:
forValue?, attachPattern?, assert?, flag?, attr, valuePattern?, default?, iterator?
children:
ANY*

All followers of KeyAttr follower must declare a forValue attribute.

Allowed values of an iterator attribute for <key_attr> element are "yes", "no" or "auto".

NodeAttr element

tag:
<node_attr>
attributes:
forValue?, attachPattern?, assert?, flag?, attr, iterator?
children:
ANY

Allowed values of an iterator attribute for <node_attr> element are "yes" or "no" since autodetection is ambiguous.
