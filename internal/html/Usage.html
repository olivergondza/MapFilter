Usage

MapFilter

To use MapFilter there is need to define Pattern (MapFilter::setPattern()) describing all possible data relations and let MapFilter do all the work related with input filtering. A filtered array can be obtained (MapFilter::getResults()). Secondary filtering outputs are two arrays: flags and assertions. Where flags (MapFilter::getFlags()) bares user defined filter achievements and asserts (MapFilter::getAsserts()) contains defined failures. For detailed explanation see Examples of use.

Pattern

MapFilter Pattern (MapFilter_Pattern) consists of groups of arguments (special king of such group may be a single argument), creating tree structure of groups as nodes and single arguments as it's leaves (MapFilter_TreePattern_Tree_Node). A new Pattern can be created from an existing object (new MapFilter_Pattern ( $pattern );), from xml file (MapFilter_Pattern::fromFile ( $url );) or xml in String (MapFilter_Pattern::load ( $xmlString )).

During the filtering process algorithm walk through the tree starting from it's root and evaluating all defined argument groups. So-called "satisfaction" of the argument group is determined by it's rules and satisfaction of it's followers. Single argument is considered "satisfied" when it's present. In result will be found all satisfied arguments from satisfied groups. flags will contain all defined flags from satisfied groups and asserts will contain all defined asserts from adjacent unsatisfied groups.

Nodes

Pattern tree structure consists of several types of nodes. There are four so-called Policy nodes (All, One, Opt and Some) that specifies data attribute relations. KeyAttr node defines key-value relations between key attribute and its possible value patterns. Attr node specifies attribute itself, it's valuePattern and pattern default value.

All

All node needs all it's followers satisfied, making it's occurrence mandatory.

Opt

Opt node hasn't any conditions. Occurrence of its followers is completely optional.

One

One node needs exactly one satisfied follower.

Some

Some node needs at least one satisfied follower.

KeyAttr

KeyAttr node differs from aforementioned node types. This node defines one attribute (attr) that must be present, depending on value of this attribute one of it's followers will be chosen to satisfy. All it's followers has to define attribute onValue containing regular expression to match attribute value. When matching follower will be satisfied, KeyAttr node will be satisfied too.

Attr

Attr node is satisfied when it's attribute occurs and it's value matches optional pattern defined by valuePattern. When this does not happen this node still can be satisfied if it's default value is sat: attribute will have that default value and node will be satisfied.
