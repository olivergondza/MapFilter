TreePattern

TreePattern Elements

Pattern tree structure consists of several types of nodes and currently one type of tree leaf.

Leaves

Attr leaf

Attr leave specifies attribute itself, its valuePattern and pattern default value. Attr leaf is satisfied when its attribute occurs in user query and its value matches the optional pattern defined by valuePattern attribute. When this does not happen this node still can be satisfied if its default value is sat: attribute will have that default value and leaf will be satisfied.

Nodes

There are four so-called Policy nodes (All, One, Opt and Some) that specifies data attribute relations. KeyAttr node defines value-key relations between key attribute and its possible value patterns.

All node

The All node needs all its followers satisfied to get satisfied itself.

Opt node

The Opt node hasn't any conditions. Occurrence of its followers is completely optional.

One node

The One node needs exactly one satisfied follower.

Some node

The Some node needs at least one satisfied follower.

KeyAttr node

KeyAttr node differs from aforementioned node types. This node defines one attribute (attr) that must be present, depending on value of this attribute one of its followers will be chosen to satisfy. All its followers can define attribute onValue containing regular expression to match attribute value. When matching follower will be satisfied, KeyAttr node will be satisfied as well.

TreePattern Common Attributes

Following attributes are shared with all TreePattern elements.

Attribute forValue

A "forValue" attribute specifies valueFilter for Key-Attr node follower. According to this filter follower to satisfy is going to be chosen.

Note:
The "filter" in the name of an attribute has nothing to do with "filter" in the name of the MapFilter class.

Attribute flag

A "flag" attribute specifies a flag to set for the node if the node get satisfied.

Attribute assert

An "assert" attribute specifies an assertion to set for the node if the node won't get satisfied even though its parent was satisfied.

TreePattern Attr leaf attributes

Attributes specific for Attr node.

Attribute default

A "default" attribute determines value that is going to be used as a key value in case attribute won't get satisfied.

Attribute valuePattern

A "valuePattern" attribute determines pattern to match for key value.

TreePattern KeyAttr node attributes

Attributes specific for KeyAttr node.

Attribute attr

An "attr" attribute specifies key attribute that is going to be used to follower determination.

Note:
Not to be confused with TreePattern Attr leaf.
