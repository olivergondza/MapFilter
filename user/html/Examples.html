Examples

Whitelists

Exclusive Whitelist

This simple example shows how to create simple whitelist allowing occurrence of just one of defined attributes. my_utility -h -v -o a.out

$query = Array (
    '-h' => NULL,
    '-v' => NULL,
    '-o' => "a.out"
);

$pattern = '
    <pattern>
      <one>
        <attr>-h</attr>
        <attr>-v</attr>
      </one>
    </pattern>
';

$filter = new MapFilter (
    MapFilter_Pattern::load ( $pattern ),
    $query
);

// Array ( "-h" => NULL );
var_dump ( $filter->parse () );

Filtered query now contains just -h option since there is nothing about -o in our pattern and -v was preceded by -h. If one run the application with reversed order of the options (my_utility -v -h -o a.out) filtered query will contains nothing more than -v option.

Simple Whitelist

Let’s assume the same query with slightly modified pattern.

$pattern = '
    <pattern>
      <some>
        <attr>-h</attr>
        <attr>-v</attr>
      </some>
    </pattern>
';

This is the way to enable occurrence of more options (including 0). The same query will now contain both -h and -v option regardless of their order.

Full Sets

Sometimes it’s useful to specify a group of attributes that all must occur. my_app -f main.c -o a.out

$pattern = '
    <pattern>
      <all>
        <attr>-f</attr>
        <attr>-o</attr>
      </all>
    </pattern>
';

$filter = new MapFilter ( MapFilter_Pattern::load ( $pattern  ) );

$query = Array (
    '-f' => "main.c",
    '-o' => "out.a"
);

$filter->setQuery ( $query );
$filter->parse ();

// Array (
//     '-f' => "main.c",
//     '-o' => "out.a"
// );
var_dump ( $filter->getResults () );

Since we have got all the arguments, they all pass our filter. Let’s try to run the binary without source file. my_app -o a.out

$query = Array (
    '-o' => "out.a"
);

$filter->setQuery ( $query );
$filter->parse ();

// Array ();
var_dump ( $filter->getResults () );

Filtered query would be empty because our pattern demand occurrence of both (all) options.
