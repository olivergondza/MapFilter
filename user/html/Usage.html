Usage

MapFilter

To use MapFilter there is need to define Pattern (MapFilter::setPattern()) describing all possible data relations and let MapFilter do all the work related with input filtering. Filtering outputs can be had by fetchResult(). This object implements the same set of interfaces as original Pattern did.

TreePattern

MapFilter TreePattern (MapFilter_TreePattern) consists of groups of arguments (special king of such group may be a single argument), creating tree structure of groups as nodes and arguments as it's leaves (MapFilter_TreePattern_Tree_Node). A new Pattern can be created from xml file (MapFilter_Pattern::fromFile ( $url );) or xml in String (MapFilter_Pattern::load ( $xmlString )).

During the filtering process algorithm walk through the tree starting from it's root and evaluating all defined argument groups. So-called "satisfaction" of the argument group is determined by it's rules and satisfaction of it's followers. Single argument is considered "satisfied" when it's present and when it matches the pattern. In result will be found all satisfied arguments from all satisfied groups. flags will contain all defined flags from satisfied groups and Asserts will contain all defined asserts from adjacent unsatisfied groups.

TreePattern Elements

Pattern tree structure consists of several types of nodes and currenty one type of tree leaf.

Attr leaf

Attr leave specifies attribute itself, it's valuePattern and pattern default value. Attr leaf is satisfied when it's attribute occurs in user query and it's value matches optional pattern defined by valuePattern attribute. When this does not happen this node still can be satisfied if it's default value is sat: attribute will have that default value and leaf will be satisfied.

There are four so-called Policy nodes (All, One, Opt and Some) that specifies data attribute relations. KeyAttr node defines value-key relations between key attribute and its possible value patterns.

All node

The All node needs all it's followers satisfied to get satisfied itself.

Opt node

The Opt node hasn't any conditions. Occurrence of its followers is completely optional.

One node

The One node needs exactly one satisfied follower.

Some node

The Some node needs at least one satisfied follower.

KeyAttr node

KeyAttr node differs from aforementioned node types. This node defines one attribute (attr) that must be present, depending on value of this attribute one of it's followers will be chosen to satisfy. All it's followers has to define attribute onValue containing regular expression to match attribute value. When matching follower will be satisfied, KeyAttr node will be satisfied too.
