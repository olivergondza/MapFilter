MapFilter TreePattern (MapFilter_TreePattern) consists of groups of arguments (special kind of such group may be a single argument), creating tree structure of groups as nodes and arguments as it's leaves. A new Pattern can be created from xml file (MapFilter_TreePattern::fromFile ( $url );) or xml in String (MapFilter_TreePattern::load ( $xmlString );).

During the filtering process algorithm walk through the tree starting from its root and evaluating all defined argument groups. So-called 'satisfaction' of the argument group is determined by its rules and satisfaction of it's followers. Single argument is considered 'satisfied' when it's present and when it matches the valuePattern of the default attribute is sat. In result will be found all satisfied arguments from all satisfied groups. Flags will contain all defined flags from satisfied groups and Asserts will contain all defined asserts from adjacent unsatisfied groups.

TreePattern Result

TreePattern currently provides three different types of results. Result object can be obtained by fetchResult(). Individual results can be obtained by getMyResultType() (i.e. $filter->fetchResult ()->getAsserts ();).

Results

Results array can be obtained by getResults (). Results array contains filtering result.

Flags

Flags array can be obtained by getFlags (). Flags array contains filtering flags.

Asserts

Asserts array can be obtained by getAsserts (). Asserts array contains filtering assertions.
